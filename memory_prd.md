## 一、产品概述

### 1.1 产品背景

当前虚拟社交 App（MyPhone）的 AI Character 具有记忆能力，但这个能力对用户是"黑箱"的：
- 用户不知道 AI 记住了什么
- 记忆错误时用户无法修正
- 记忆分类混乱，用户无法理解

### 1.2 产品目标

让用户**看见、理解、干预** AI 的记忆系统，建立用户信任，提升产品体验。

### 1.3 核心价值

| 价值点 | 说明 |
|--------|------|
| **透明** | 用户可以像看日记一样查看 AI 记忆 |
| **可控** | 用户可以编辑、删除、新增记忆 |
| **可理解** | 记忆按类型/来源/角色清晰分类 |

## 四、UI 设计（治愈系风格）

### 4.1 设计理念

**核心美学**：日记本、手账、治愈系

- 像真实的纸质日记本一样温暖
- 每条记忆像一张便签纸
- 柔和的色调，舒适的阅读体验
- 手写字体，文艺感

### 4.2 全局样式规范

#### 时间分组逻辑

- 今天：日期 == 今天
- 昨天：日期 == 昨天
- 本周：日期在本周内但非今天/昨天
- 上周：日期在上周内
- 更早：日期在上周之前

## 六、API 设计

### 6.1 接口列表

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/memories` | 获取记忆列表 |
| `GET` | `/api/memories/:id` | 获取单条记忆 |
| `POST` | `/api/memories` | 创建记忆 |
| `PATCH` | `/api/memories/:id` | 更新记忆 |
| `DELETE` | `/api/memories/:id` | 删除记忆（软删除） |
| `GET` | `/api/memories/stats` | 获取统计 |
| `POST` | `/api/memories/export` | 导出记忆 |
| `POST` | `/api/memories/import` | 导入记忆 |

### 6.2 通用规范

#### 鉴权
- 所有接口需携带用户鉴权 Token（通过 `Authorization: Bearer <token>` header）
- 接口自动根据 Token 解析 `userId`，用户只能操作自己的记忆

#### 分页
- `pageSize` 最大限制：100
- 默认 `pageSize`：20

#### 错误响应格式
```json
{
  "error": {
    "code": "MEMORY_NOT_FOUND" | "UNAUTHORIZED" | "VALIDATION_ERROR" | "INTERNAL_ERROR",
    "message": "错误描述"
  }
}
```

#### 删除策略
- **软删除**：将 `enabled` 设为 `false`，不物理删除数据
- 删除后的记忆不在列表中展示，但可通过 ID 恢复

### 6.3 请求/响应格式

#### GET /api/memories

**Query 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `characterId` | string | 否 | 按角色筛选 |
| `app` | string | 否 | 按 App 筛选 |
| `type` | string | 否 | 按类型筛选 |
| `scope` | string | 否 | 可见范围 |
| `keyword` | string | 否 | 搜索关键词 |
| `isPinned` | boolean | 否 | 是否置顶 |
| `page` | number | 否 | 页码，默认 1 |
| `pageSize` | number | 否 | 每页条数，默认 20 |

**排序规则**：
- 默认排序：`isPinned DESC, weight DESC, updatedAt DESC`
- 置顶记忆始终在最前面

| 情绪状态 | `emotion_pattern` | 用户情绪记录 | 66 | `我今天...` |
| 约定承诺 | `promise` | 用户希望记住的事项 | 76 | `(?:记得|提醒我|别忘了)(...)` |

### B. 可见范围定义（与现有系统一致）

| 范围 | Key | 说明 | AI 优先级 |
|------|-----|------|----------|
| 私有 | `private` | 仅当前角色可见 | 50 |
| 公有 | `public` | 所有角色可见 | 50 |
| 系统 | `system` | 用户主动设定，最高优先级 | 100 |

### C. 作用域定义（与现有系统一致）

| 作用域 | Key | 说明 |
|--------|-----|------|
| 角色私有 | `character_private` | 仅特定角色可见 |
| 用户全局 | `user_global` | 用户所有角色共享 |
| App私有 | `app_private` | 仅特定 App 可见 |
| 公有 | `public` | 所有 App 和角色可见 |

### D. 来源 App 定义（与现有系统一致）

| App | Key | 说明 |
|-----|-----|------|
| 微信 | `wechat` | 微信对话 |
| 通讯录 | `contacts` | 通讯录 App |
| 系统 | `system` | 系统设定 |
| 朋友圈 | `moments` | 朋友圈动态 |

### E. 敏感度定义（与现有系统一致）

| 敏感度 | Key | 说明 |
|--------|-----|------|
